1. Parameterized Queries

SQL queries utilize place-markers `?` separating data from command logic. Attempts to inject SQL payloads (e.g. `' OR '1'='1`) are interpreted purely as string literals, rendering SQL Injection impossible.

2. Bcrypt Hashing

Passwords are processed through blowfish-based salting and hashing with a workload cost of 12. Generates unique hashes even for identical inputs, blocking precomputed rainbow table attacks.

3. CSRF Defense

Forms include a hidden cryptographically random CSRF token checked server-side before processing POST requests. Sessions use `SameSite=Lax` headers to block cross-site request forgery.

4. Secure Headers

Includes HTTP security responses: `X-Frame-Options` blocks clickjacking, `X-Content-Type-Options` blocks MIME sniffing, and `Content-Security-Policy` limits script executions (mitigating XSS).
